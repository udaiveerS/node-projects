Compiler Overview

We chose to write a procedural implementation of a TypeScript compiler. TypeScript is simply Javascript, but with types and advanced class-related features. The class-related features are not present in our implementation due to the fact that it is a procedural-style implementation.

Our implementation includes support for arithmetic operations, control structures in the form of an if statement and a while statement, functions, and print statements capable of printing strings,numbers, and booleans. Numerical types are as they are in TypeScript: they are all floating point values.

Notes about where our implementation deviates from TypeScript:

  1. Lexical scoping is not currently implemented. The function parse lexical scopes, but they variables are declared statically
  2. Type checking is not fully implemented, but types of variables must still be declared (one of the features of TypeScript).
  3. Our language’s string type is called “char”, whereas TypeScript calls it “string”.
  4. TypeScript’s print function - ‘console.log’ - is called ‘println’ in our implementation. The print function is overloaded and can infer the type of the object being printed (boolean, char, number).
  5. Our implementation parses function parameters, but function parameter code generation is not implemented due to lack of time.
  6. String operations are not implemented in our language. They exist only to be assigned to variables and printed by our “println” function.
